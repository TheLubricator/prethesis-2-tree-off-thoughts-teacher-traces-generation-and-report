3. **Efficiency gains**: Lower temperature + better prompts = fewer wasted API calls

## 🧪 What is CodeAct?

**CodeAct** is a pattern where the LLM generates executable Python code instead of natural language reasoning:

### Traditional ToT (Text-based)
```
Thought: "Let's try 6 * 9 = 54, then 54 + 10 = 64, then 64 - 9 = 55"
Evaluation: "This doesn't equal 24, so this path fails"
```

### CodeAct ToT (Code-based) ⭐
```python
# Generated by Gemma 3 27B
result = 6 * 9  # = 54
result = result + 10  # = 64
result = result - 9  # = 55
# Automatically evaluated: 55 ≠ 24, path pruned
```

**Benefits:**
- ✅ **Perfect accuracy**: No arithmetic errors
- ✅ **Self-validating**: Code execution provides immediate feedback
- ✅ **Structured output**: Easier to parse and verify
- ✅ **Debugging friendly**: Can inspect intermediate values

📖 **Learn more**: [`CODEACT_PURPOSE_AND_ARCHITECTURE.md`](CODEACT_PURPOSE_AND_ARCHITECTURE.md)

## 🎮 Game of 24

Given four numbers, use +, -, ×, ÷ to make 24. Each number must be used exactly once.

|--------|----------|-----------|

## 🔑 Key Innovation: "Try BOTH Orders" Prompt

### The Problem
Original prompt allowed the LLM to develop a bias:
- ✅ Proposed: `6 × 9 = 54`
- ❌ Never proposed: `9 × 10 = 90` (correct path!)
- **Pattern**: Model learned "smaller × larger" preference

### The Solution
Enhanced prompt with explicit instruction:

```python
PROPOSE_PROMPT_CODEACT = """
...
🔧 TRY BOTH ORDERS:
- For multiplication/addition: try BOTH a×b AND b×a
- For subtraction/division: try BOTH a-b AND b-a, a÷b AND b÷a
...
"""
```

### The Impact
- [6,9,9,10]: **FAILED** → **SOLVED** ✅
- API calls: **-31.6%** (fewer wasted explorations)
- Success rate: **+20%** improvement

📖 **Full details**: [`PROMPT_FIX_BOTH_ORDERS.md`](PROMPT_FIX_BOTH_ORDERS.md)

## 🛠️ Technical Implementation

### Model Configuration
- **Model**: Gemma 3 27B (via Google AI Studio)
- **Temperature**: 0.7 (balanced creativity + focus)
- **Selection**: Probabilistic (explores diverse paths)
- **Rate Limiting**: 3.5s delay (free tier: 14k requests/day)

### Tree Search Algorithm
1. **Propose**: Generate 3-5 next operations using CodeAct
2. **Evaluate**: Execute code, check if result = 24
3. **Prune**: Remove invalid/duplicate states
4. **Select**: Keep top-k states (k=5) using value function
5. **Backtrack**: If dead end, return to previous state

### Optimizations
- ✅ Duplicate detection (avoid `6+9` and `9+6` redundancy)
- ✅ Commutative operation handling
- ✅ Confidence-based pruning
- ✅ Early termination on solution found
